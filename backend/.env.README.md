# Assignment Grader - Environment Configuration

## Setup Instructions

1. Copy `.env.example` to `.env`:
   ```bash
   cp .env.example .env
   ```

2. Update the following values in `.env`:

   - **MONGODB_URI**: Your MongoDB Atlas connection string
     - Format: `mongodb+srv://username:password@cluster.mongodb.net/database-name`
     - Get this from MongoDB Atlas → Connect → Connect your application
   
   - **JWT_SECRET**: A strong random secret key for JWT token signing
     - Generate using: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`
   
   - **DB_TYPE**: Choose between `json` or `mongodb`
     - Use `mongodb` for production
     - Use `json` for local development without database

3. Never commit `.env` file to Git (it's in .gitignore)

## MongoDB Atlas Setup

1. Create account at https://cloud.mongodb.com
2. Create a new cluster (free tier available)
3. Add database user with username/password
4. Whitelist your IP address (or use 0.0.0.0/0 for development)
5. Get connection string and replace in MONGODB_URI

## Security Notes

- Keep `.env` file secure and never share
- Use different JWT_SECRET for production
- Rotate secrets regularly
- Use environment variables in production deployment
